# Улучшение изображений ML-моделью

Веб-приложение, которое улучшает фотографии по яркости, контрасту и насыщенности.
ML-модель работает **полностью в браузере** – изображения никуда не загружаются.

**Демо:** https://sskimdp.github.io/PracticeVK_ML-model/
**Код:** https://github.com/sskimdp/PracticeVK_ML-model

## Идея решения

Ключевое архитектурное решение: модель **не генерирует пиксели**, а решает задачу
**регрессии** – по уменьшенной копии изображения (128×128) предсказывает три
коэффициента коррекции (яркость, контраст, насыщенность). Вспомогательный
детерминированный алгоритм применяет эти коэффициенты к полному изображению.

Это даёт крошечную модель (112 КБ) и обработку даже 15-мегапиксельного снимка
примерно за секунду, при этом всё работает на стороне клиента.

Модель обучена приводить изображение к **эталонной тональной и цветовой
статистике** (мягкие авто-уровни по гистограмме плюс цель по насыщенности), а не отменять искажение. Поэтому нормальные фотографии получают аккуратное
улучшение и не портятся, а плохие – заметную коррекцию.

## Возможности

- Загрузка перетаскиванием или выбором файла
- Форматы: JPG, PNG, HEIC, BMP
- Изображения до 15 Мпк
- Асинхронная обработка в Web Worker – интерфейс не блокируется
- Прогресс по стадиям и отмена задачи
- Сравнение «до/после» слайдером
- Ручная тонкая настройка поверх авто-результата: слайдеры яркости, контраста,
  насыщенности и общей интенсивности, live-превью, сброс к авто

## Как это работает

```
UI (главный поток)
   декодирование HEIC -> PNG (heic2any требует window)
   вызовы API + подписка на событие statusChange
        |
        v
ImageEnhancerAPI (фасад)
   createTask · getStatus · cancelTask · getResult · reapplyTask · getPreview
        |  postMessage (transferable)
        v
Web Worker (фоновый поток)
   1. декодирование JPG/PNG/BMP + проверка 15 Мпк
   2. thumbnail 128x128 -> tensor
   3. инференс TF.js -> { brightness, contrast, saturation }
   4. применение коррекции к полному изображению (чанками)
   5. кодирование результата (PNG)
```

Подробнее – в [ТЗ.md](ТЗ.md). Отчёт об оценке качества – в [eval/report.md](eval/report.md).

## Соответствие требованиям

| Требование | Лимит | Факт |
|---|---|---|
| Работа во всех массовых браузерах | – | стандартные API + фолбэк бэкендов WebGL -> CPU |
| Объём кода | ≤ 10 МБ | 3.3 МБ |
| Разрешение | до 15 Мпк | проверка + понятная ошибка при превышении |
| Максимальное время | ≤ 30 с | ~1 с на 15 Мпк |
| Среднее время | ~5 с | десятки мс – ~1 с |
| Форматы | JPG, PNG, HEIC, BMP | все поддержаны |
| Асинхронный режим | – | Web Worker, UI свободен |

## Качество

Оценка на пуле эталонных изображений (5 категорий искажений, 40 фото, метрики
PSNR/SSIM относительно оригинала):

| Категория | PSNR до | PSNR после |
|---|---|---|
| Тёмные | 12.4 | 23.8 |
| Пересвеченные | 14.8 | 16.7 |
| Низкоконтрастные | 21.4 | 24.9 |
| Смешанные | 14.8 | 22.7 |

Уже хорошие фотографии модель почти не меняет (среднее изменение пикселя ~4 %).
Полный отчёт, граничные кейсы (ч/б, очень маленькие, уже хорошие) и состав пула –
в [eval/report.md](eval/report.md).

## Обучение модели

Модель обучается self-supervised: реальные фотографии скачиваются скриптом, для
каждой аналитически вычисляются «идеальные» факторы коррекции, и модель учится их
предсказывать. Требуется Python 3.9 (TensorFlow не поддерживает 3.13+); версии
зависимостей подобраны как совместимый набор в `training/requirements.txt`.

```bash
cd training
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

.venv/bin/python train.py        # обучение -> model.h5
.venv/bin/python export.py       # конвертация в public/model/ (TF.js, uint16)
```

Оценка качества:

```bash
.venv/bin/python ../eval/evaluate.py   # отчёт -> eval/report.md
```

## Инженерные решения

- **Асинхронность.** Декодирование, инференс и попиксельная коррекция вынесены в
  Web Worker; коррекция идёт чанками с проверкой отмены, поэтому интерфейс не
  замирает даже на 15 Мпк.
- **Размер.** TF.js (~1.9 МБ) и декодер HEIC (~1.35 МБ) – ленивые чанки, модель
  квантована до uint16 (112 КБ); стартовая загрузка ~28 КБ.
- **Память.** Освобождение тензоров, переиспользование декодированного оригинала
  для ручной перенастройки, очистка данных прошлых задач; на повторных прогонах
  15 Мпк утечек нет.
- **Кроссбраузерность.** Только стандартные API; фолбэк бэкендов TF.js
  WebGL -> CPU; HEIC декодируется в главном потоке, так как heic2any требует window.
- **Отказоустойчивость.** Любая ошибка переводит задачу в статус error без
  падения приложения; при сбое загрузки модели – повторная попытка и мягкий
  фолбэк, при котором изображение не портится.
